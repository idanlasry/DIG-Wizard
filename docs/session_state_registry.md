# DIG Analytics ‚Äî Session State Registry

| `st.session_state.*` | What it stores | Initialized by | Read by (LLM agent) |
|---|---|---|---|
| `initialized` | Boot flag ‚Äî prevents re-init on rerun | app.py startup | None |
| `stage` | Current pipeline stage string (`START ‚Üí AUDIT ‚Üí RESEARCH ‚Üí ANALYSIS ‚Üí DASHBOARD`) | app.py startup (`"START"`) | PM Agent (as `CURRENT_STAGE`) |
| `raw_data` | Uploaded CSV as pandas DataFrame | app.py ‚Äî after `pd.read_csv()` | None ‚Äî LLM never sees raw data |
| `metadata` | Profiler output dict ‚Äî schema, nulls, outliers, distributions | app.py ‚Äî after `get_dataset_profile()` | DE Agent, Researcher Agent, PM Agent |
| `de_findings` | DE Agent output ‚Äî quality score, issues, outliers JSON | AUDIT stage ‚Äî DE Agent call | PM Agent, Researcher Agent |
| `de_running` | Bool ‚Äî whether DE Agent is currently executing (spinner lock) | app.py startup (`False`) | None ‚Äî internal UI flag |
| `pm_ready` | Bool ‚Äî whether PM Agent approved stage transition out of AUDIT | PM gate call at AUDIT | AUDIT stage ‚Äî gates PROCEED button |
| `pm_summaries` | Accumulated list of all PM `user_message` strings across all gate calls | app.py startup (`[]`) | None ‚Äî rendered in PM SUMMARY / PM LOG tabs |
| `pm_final_summary` | PM Agent final synthesis string produced at DASHBOARD entry | DASHBOARD stage ‚Äî PM gate call | BI Agent (as `pm_final_summary` param) |
| `report_view` | Active tab key in Living Report panel | app.py startup (`"metadata"`) | None ‚Äî app.py content router only |
| `history_logs` | List of timestamped log dicts for the terminal panel | app.py startup (one system entry) | None ‚Äî display only |
| `master_report` | Legacy living markdown placeholder ‚Äî fallback render if no view matches | app.py startup (placeholder string) | None |
| `research_paths` | 5 research path options generated by Researcher Agent | AUDIT stage ‚Äî Researcher Agent call | None ‚Äî user selects from UI |
| `primary_metric` | Dict identifying the primary binary KPI column, or `None` if not detected | AUDIT stage ‚Äî Researcher Agent call | None ‚Äî used in HTML report builder |
| `current_path` | The single research path the user selected | RESEARCH stage ‚Äî user click | DA Agent (as `path`), PM Agent (as `selected_path`) |
| `tool_result` | List of raw output dicts from Starter Kit functions (one per tool instruction) | ANALYSIS stage ‚Äî Switchboard execution | DA Agent (as `tool_results`) |
| `da_findings` | DA Agent interpretation of tool results for the current path | ANALYSIS stage ‚Äî DA Agent call | None ‚Äî reset to `None` on next path; persisted in `analysis_results` |
| `analysis_results` | List of all completed path dicts ‚Äî accumulates across the loop | app.py startup (`[]`) | PM Agent (as `previous_findings`), BI Agent, Synthesis Agent |
| `cross_path_summary` | Pre-computed cross-path signals dict ‚Äî most-cited columns, shared stat labels, potential overlaps | app.py startup (`{}`); updated after each path completes | DA Agent (paths 2+, as `PRIOR PATH SIGNALS`), Synthesis Agent (as `CROSS-PATH SIGNALS`) |
| `chart_configs` | BI Developer Agent output ‚Äî KPIs + Plotly chart configs + narrative | DASHBOARD stage ‚Äî BI Agent call | Synthesis Agent (as `chart_configs`) |
| `synthesis` | Synthesis Agent output ‚Äî cross-path narrative + recommendations | DASHBOARD stage ‚Äî Synthesis Agent call | None ‚Äî rendered in DASHBOARD and HTML report |
| `api_call_count` | Running count of LLM API calls made this session | app.py startup (`0`) | None ‚Äî footer display only |
| `total_input_tokens` | Cumulative input tokens across all API calls | app.py startup (`0`) | None ‚Äî footer display only |
| `total_output_tokens` | Cumulative output tokens across all API calls | app.py startup (`0`) | None ‚Äî footer display only |
| `estimated_cost_usd` | Cumulative estimated USD cost of all API calls | app.py startup (`0.0`) | None ‚Äî footer display only |
| `user_interest_choice` | User's pre-research choice: `"none"` (let researcher run freely) or `"yes"` (user has a specific interest) | app.py startup (`"none"`) | None ‚Äî UI flag only |
| `user_interest_text` | Free-text interest the user typed before research; empty string if none | app.py startup (`""`) | Researcher Agent ‚Äî injected as `USER_INTEREST` section in context |
| `user_interest_path` | Researcher Agent's response to the user's interest ‚Äî a path dict if feasible, or a dict with a `feasibility_note` if not; `None` if no interest was given | AUDIT stage ‚Äî Researcher Agent call | None ‚Äî rendered as a card in RESEARCH stage |

## Sub-Attributes

### `metadata`
Produced by `profiler.py` ‚Äî `get_dataset_profile()`

| Key | Type | Description |
|---|---|---|
| `shape` | dict | `{"rows": int, "cols": int}` |
| `full_row_count` | int | Total rows before any sampling |
| `duplicate_rows` | int | Count of exact duplicate rows |
| `columns` | list[dict] | One entry per column ‚Äî see below |
| `numeric_summary` | dict | Keyed by column name ‚Äî see below |
| `categorical_summary` | dict | Keyed by column name ‚Äî see below |
| `datetime_flags` | dict | Keyed by column name ‚Äî bool, whether column parses as datetime |

**`columns` entry:**
| Key | Type | Description |
|---|---|---|
| `name` | str | Column name |
| `dtype` | str | Pandas dtype string |
| `null_count` | int | Number of null values |
| `null_pct` | float | Null percentage |
| `sample_values` | list[str] | Up to 5 unique values |

**`numeric_summary` entry (per column):**
| Key | Type | Description |
|---|---|---|
| `mean` | float | Column mean |
| `std` | float | Standard deviation |
| `min` | float | Minimum value |
| `max` | float | Maximum value |
| `outliers_iqr_count` | int | Count of IQR outliers |
| `outliers_zscore_count` | int | Count of Z-score outliers |
| `outlier_sample_values` | list[float] | Up to 5 outlier values |

**`categorical_summary` entry (per column):**
| Key | Type | Description |
|---|---|---|
| `unique_count` | int | Number of distinct values |
| `top_values` | dict | Top 5 values with counts `{value: count}` |

---

### `de_findings`
Produced by DE Agent ‚Äî validated by Pydantic

| Key | Type | Description |
|---|---|---|
| `quality_score` | int | 1‚Äì10 rating |
| `quality_score_reason` | str | One sentence explanation |
| `dataset_summary` | dict | `{total_rows, total_columns, is_sample}` |
| `columns` | list[dict] | `{name, type, sample_value}` per column |
| `quality_issues` | list[dict] | `{issue, detail, affected_column}` ‚Äî empty if none |
| `outliers` | list[dict] | `{column, value, reason}` ‚Äî empty if none |

---

### `primary_metric`
Produced by Researcher Agent ‚Äî `None` if no binary column detected

| Key | Type | Description |
|---|---|---|
| `label` | str | Human-readable label (e.g. `"Churn"`) |
| `column` | str | Source column name |
| `rate_pct` | float | Mean √ó 100 ‚Äî the positive-class rate |

---

### `history_logs` entry
| Key | Type | Description |
|---|---|---|
| `time` | str | Timestamp `HH:MM:SS` |
| `msg` | str | Log message |
| `type` | str | `"system"`, `"info"`, or `"error"` |

---

### `analysis_results` entry
Accumulates after each completed research path

| Key | Type | Description |
|---|---|---|
| `path` | dict | The selected research path from Researcher Agent |
| `tool_result` | list[dict] | Raw Starter Kit outputs (one per tool instruction) |
| `da_findings` | dict | DA Agent interpretation ‚Äî see `da_findings` below |

---

### `da_findings`
Produced by DA Agent ‚Äî validated by Pydantic

| Key | Type | Description |
|---|---|---|
| `headline` | str | One-sentence finding summary |
| `key_insights` | list[str] | 3‚Äì5 bullet insights |
| `supporting_stats` | list[dict] | `{label, value, context}` per stat |
| `caveats` | str | Limitations or data quality warnings (may be empty) |
| `recommended_viz` | str | Suggested chart type |
| `viz_rationale` | str | One sentence explaining the chart choice |

---

### `chart_configs`
Produced by BI Developer Agent ‚Äî validated by Pydantic

| Key | Type | Description |
|---|---|---|
| `dashboard_narrative` | str | One-paragraph executive framing |
| `kpis` | list[dict] | `{label, value, delta, context}` per KPI |
| `charts` | list[dict] | Plotly-compatible chart configs ‚Äî see below |

**`charts` entry:**
| Key | Type | Description |
|---|---|---|
| `chart_type` | str | `"bar"`, `"line"`, or `"heatmap"` |
| `title` | str | Chart title |
| `x_label` | str | X-axis label |
| `y_label` | str | Y-axis label |
| `source_path_index` | int | 0-based index into `analysis_results` |
| `source_tool` | str | Name of the starter kit tool whose output to render |
| `source_col` | str \| null | Column to disambiguate multi-run tools; null = aggregate all runs |
| `source_path` | str | Research path title (display label) |
| `explanation` | str | Caption text explaining the chart |

---

### `user_interest_path`
Produced by Researcher Agent ‚Äî validated by `UserInterestPath` Pydantic model. `None` if no user interest was provided.

| Key | Type | Description |
|---|---|---|
| `title` | str | Short path name (3‚Äì6 words) |
| `question` | str | The business question the user wanted answered |
| `rationale` | str | 2‚Äì4 sentences: what will be checked, why it matters, what insight the tools will surface |
| `tool_instructions` | list[dict] \| null | Ordered tool calls (2‚Äì5), same `{tool, params}` schema as regular paths; `null` when not feasible |
| `feasibility_note` | str \| null | Explanation of why the dataset or available tools cannot answer the interest; `null` when feasible |

---

### `synthesis`
Produced by Synthesis Agent ‚Äî validated by Pydantic

| Key | Type | Description |
|---|---|---|
| `narrative` | str | Cross-path strategic narrative paragraph |
| `recommendations` | list[str] | Ordered actionable recommendations |
